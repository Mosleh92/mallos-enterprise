# Security & Reporting Policy

## 🔒 Reporting a Vulnerability

If you discover a security issue in this codebase, please **do NOT** open a public issue. Instead, contact the maintainer directly so the issue can be addressed before public disclosure.

- **Email:** moslehmohammad2@gmail.com
- **Subject line:** `[SECURITY] <repo-name> — <short description>`

Expected response time: **within 72 hours**.

## 🛡️ Scope

This policy covers all repositories under the [@Mosleh92](https://github.com/Mosleh92) account.

## ✅ What Counts

- Authentication / authorization bypasses
- Injection vulnerabilities (SQL, command, prompt, etc.)
- Sensitive data exposure
- Insecure deserialization
- Server-side request forgery
- Logic flaws with security impact

## ❌ Out of Scope

- Self-XSS that requires the victim to paste code into their console
- Issues in third-party dependencies (report those upstream)
- Theoretical vulnerabilities without a working proof-of-concept

## 🤝 Responsible Disclosure

I commit to:

1. Acknowledging your report within 72 hours
2. Investigating and validating the issue promptly
3. Crediting you in the fix commit / release notes (unless you prefer anonymity)
4. Not pursuing legal action against good-faith researchers who follow this policy
